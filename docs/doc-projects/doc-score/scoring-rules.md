# 评分规则

## 核心原则

1. **证据优先**：每个扣分点必须绑定原文证据，没有证据不得重扣
2. **只评文档质量**：不评价业务方案好坏
3. **先判断类型，再选标准**：先识别文档类型，再应用对应维度评分
4. **区分问题等级**：区分"阻断任务""显著影响效率""轻微规范问题"
5. **保持分数可校准**：同类问题扣相近分数

## 评分结构（100 分）

总分 = 通用维度（60 分） + 专属维度（40 分）

### 通用维度（60 分，所有文档通用）

| 维度 | 分值 | 关注点 |
| --- | ---: | --- |
| A. 内容准确性与可信度 | 25 | 术语、参数、路径、链接是否正确 |
| B. 清晰可理解与一致性 | 15 | 结构、表达、格式是否统一 |
| C. 写作规范与文字质量 | 10 | 标题、标点、列表、代码块规范 |
| D. 核心目标达成度 | 10 | 读者能否完成文档应帮助完成的任务 |

### 专属维度（40 分，按文档类型不同）

| 文档类型 | E1 维度（20 分） | E2 维度（20 分） |
| --- | --- | --- |
| 操作指南型 | 步骤完整性 | 可执行性 |
| 概念解释型 | 概念清晰度 | 决策支持与衔接 |
| API/技术参考型 | 接口完整性 | 示例可用性 |
| 规则条款型 | 规则可执行性 | 边界与验收清晰度 |
| FAQ/排障型 | 问题覆盖度 | 解决方案闭环 |
| 流程总览型 | 流程完整性 | 导航效率 |

## 文档类型识别

评分前先识别文档类型，根据内容特征判断，不仅依赖目录路径：

| 文档类型 | 内容特征 | 核心目标 |
| --- | --- | --- |
| 操作指南型 | 有前提条件 + 分步操作 + 截图 | 让读者按步骤完成操作 |
| 概念解释型 | 解释定义、原理、关系 | 让读者理解概念并做出决策 |
| API/技术参考型 | 接口列表、参数表、代码示例 | 让开发者正确使用接口 |
| 规则条款型 | 条款编号、处罚措施 | 让读者明确权利义务 |
| FAQ/排障型 | 问题→排查→解决方案 | 帮读者定位并解决问题 |
| 流程总览型 | 完整流程、大量链接 | 让读者了解全貌并定位下一步 |

## 总分锚点

| 分数段 | 含义 |
| --- | --- |
| 90～100 | 高质量标杆，只有轻微优化项 |
| 80～89 | 基本可用，存在明确优化空间 |
| 70～79 | 可部分使用，关键闭环有明显缺口 |
| 60～69 | 风险较高，依赖经验才能完成任务 |
| `<60` | 未达成核心目标 |
