# 快速开始

这是我的第一篇 Docs as Code 文档。

## 什么是 Docs as Code

Docs as Code 是一种文档管理方法论，核心思想是：

- 用 **Markdown** 写文档（不用 Word）
- 用 **Git** 做版本控制（不用手动备份）
- 用 **CI/CD** 自动构建发布（不用手动上传）
- 用 **代码评审** 流程审核文档变更（PR / MR）

## 为什么用这种方式

| 传统方式 | Docs as Code |
| ---------- | -------------- |
| Word/Confluence 手写 | Markdown + Git |
| 手动发布 | CI/CD 自动部署 |
| 版本混乱 | Git 历史记录清晰 |
| 难以多人协作 | PR 评审 + 分支管理 |
