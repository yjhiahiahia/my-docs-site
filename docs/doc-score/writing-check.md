# 写作规范检查

除了质量评分，我们还有一套独立的写作规范检查，专注于格式、表达和结构规范，不评价内容正确性。

## 检查维度总览

| 维度 | 检查重点 |
| --- | --- |
| 标题规范 | 层级、跳级、重复、标点 |
| 段落与语句 | 长句、歧义、冗余、逻辑 |
| 列表规范 | 有序/无序、层级、一致性 |
| 表格规范 | 表头、空单元格、一致性 |
| 图片规范 | 标题、截图范围、敏感信息 |
| 说明/注意组件 | 必须使用指定组件格式 |
| 代码块规范 | 语言标识、缩进 |
| 链接规范 | 链接文本、相对路径、裸链接 |
| 术语与缩略语 | 首次解释、大小写统一 |

## 标题规范

检查项：

1. 标题层级是否超过 4 级
2. 是否存在跳级（如 H1 → H3）
3. 同一层级标题结构是否一致
4. 是否存在重复标题
5. 标题是否包含标点符号
6. 是否存在"父标题只有一个子标题"的情况

## 段落与语句

检查项：

1. 是否存在长句（一逗到底）
2. 是否存在歧义表达或双重否定
3. 代词指代是否明确
4. 是否存在啰嗦冗长的表达
5. 是否存在逻辑混乱（因果、条件、转折关系不清晰）
6. 同一文档中是否重复表达同一内容
7. 是否使用主动时态

## 说明/注意组件（高优先级）

这是强校验规则。"说明"和"注意"必须使用指定的 blockquote 组件，不允许直接写在正文中。

正确格式：

- 说明类用绿色组件（blockquote-green）
- 注意类用蓝色组件（blockquote-highlight）

违规情况：

- 使用"说明：/注意："但未使用组件
- 使用错误颜色的组件
- 未换行或列表格式错误

## 输出格式

检查结果按维度分类输出，每个问题必须指出：

- 问题类型
- 问题位置
- 不符合的规范点

最后列出 3 个最严重的问题（Top3）。
