# 文档质量评分系统

## 项目背景

小米 IoT 开发者平台已有 50+ 篇文档，文档 3.0 改版后内容规模扩大，但缺少统一的质量评估标准，难以及时识别结构不清、步骤缺失、表达不规范等问题。

## 我做了什么

从 0 设计了一套「通用维度 + 文档类型专属维度」的评分体系，并通过 AI + 脚本实现批量评估：

1. 梳理评估维度，设计证据型评分规则
2. 按文档类型（操作指南、概念说明、API 参考、FAQ 等）设置差异化标准
3. 设计 AI 检查 Prompt，要求基于原文输出问题依据和修改建议
4. 编写 Python 脚本，支持单篇评分和批量评分
5. 集成到 CI/CD 流水线，文档提交时自动质量检查
